Another take on - "Correlation does not imply causation". So what does?

2019/02/14

Intro

In a recent post I gave an intuitive demonstration of what’s causal inference and how it’s different than classic ML.

After receiving some feedback I realize that while the post was easy to digest, some confusion remains. In this post I’ll delve a bit deeper into causal inference while building on the example presented.

The “do” operator

The fundamental concept of causal inference is that of the “do” operator.

If we observe a variable \(X\) taking some specific value \(x\) we write \(X=x\). If however we “force” or set \(X\) to some value \(x\) then we write \(do(X=x)\).

In classic ML we have some target variable \(Y\) for which we’d like to predict the expected outcome given some feature set \(X\) was observed:

\[\mathbb{E}(Y|X=x)\]

In causal inference however we’re interested in predicting the expeted outcome of \(Y\) given we set \(X\) to some value \(x\):

\[\mathbb{E}(Y|do(X=x))\] The difference between Causal Inference and ML boils down (in a nutshell) to the 2 equations above.

In order to understand why they are fundamentally different let’s go back to the example presented in the last post.

A typical data science task

Imagine we’re tasked by the marketing team to predict what would be the number of sales for a range of mkt values. We have at our disposal historic records of marketing spend (mkt), visits to our website (visits), sales, and competition index (comp).

We’ll simulate a dataset using a set of equations (also called structural equations):

\[1) \, sales = \beta_1vists + \beta_2comp + \epsilon_1\]

\[2) \, vists = \beta_3mkt + \epsilon_2\]

\[3) \, mkt = \beta_4comp + \epsilon_3\]

\[4) \, comp = \epsilon_4\]

with \(\{\beta_1, \beta_2, \beta_3\, \beta_4\} = \{0.3, -0.9, 0.5, 0.6\}\) and \(\epsilon_i \sim N(0,1)\).

Equation 3 above reflects the fact that the marketing team would usually raise mkt in response to higher competition.

We can think of the quetions above as some data generating process (DGP) underlying our simulated dataset. We say that the data generated by this process is distributed \(\{Y,X\} \sim DGP_1\)

Looking at the plot above one could erroneously arrive at the conclusion that raising marketing spend reduces sales.

This is however incorrect. If we were to set mkt then the data generating process would change: Setting the mkt value would mean we should delete equation 3 above (\(mkt = \beta_4comp + \epsilon_3\)) as mkt is no longer a function of competition. After setting mkt the rest of the variables are generated according to the remaining equations.

Let’s label the data generating process described above as \(DGP_2\).

We can see that the expected sales for historicaly observed mkt values (shown in the first plot) is vastly different then the those we get when set set the mkt values.

The challenge with Causal inference

The reason why we need new algorithms and tools to predict \(\mathbb{E}(Y|do(X=x))\) stems from the fact we need to make predictions using data \(\{Y,X\}\) generated by \(DGP_1\) on a quantity that is generated under \(DGP_2\). That’s very much unlike classic ML where we predict \(\mathbb{E}(Y|X=x)\) which is generated under \(DGP_1\) using observations generated by the same process.

A further complication that arises from this situation is that we never have ground truth to test our models against. In classic ML no matter how we build predictive models \(f(X)\), we can always benchmark their accuracy and understand how well they are doing in general and compared to one another. That’s because we have samples \(\{Y_i, \hat{Y}_i\}\) (both elements are generated under the same DGP) which can be used to construct unbiased estimators of the expected error \(\mathbb{E}(Y-\hat{Y})\).
